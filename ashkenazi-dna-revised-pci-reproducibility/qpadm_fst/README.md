# qpAdm and FST audit trail

This directory preserves the author-generated qpAdm and autosomal FST analyses retained in the revised manuscript.

## Original environment

- Platform: IllustrativeDNA Admix Lab web interface
- Dataset: Human Origins v62.0
- qpAdm setting: `allsnps = TRUE`
- FST setting: `Adjust Pseudohaploids = NO`
- qpAdm target: `Jew_Ashkenazi.HO`

The analyses were not generated by local ADMIXTOOLS shell commands. The files here preserve the original web-platform workflow rather than presenting a later command-line reconstruction as the original analysis.

## qpAdm models

Three models hold Lebanese and Russian sources constant while changing the Italian source:

1. `Italian_South.HO + Lebanese.HO + Russian.DG`
2. `Italian_North.HO + Lebanese.HO + Russian.DG`
3. `Italian_Central.HO + Lebanese.HO + Russian.DG`

Exact right-set labels and settings are in `inputs/` and `models/`. Copied raw outputs are in `raw_outputs/qpadm/`, and the manuscript table is `tables/table9_qpadm_results.csv`.

## FST comparisons

The FST comparison uses `Jew_Ashkenazi.HO` as the target and the declared Mediterranean and Jewish comparator populations. Exact comparisons are in `models/fst_comparisons.tsv`; copied output is in `raw_outputs/fst/`; manuscript tables are `tables/table10_fst_results.csv` and `tables/table11_fst_calibration.csv`.

## Reproduction

1. Open the same web platform and select Human Origins v62.0.
2. Enter the exact labels in `inputs/`.
3. Apply the settings above.
4. Compare the returned output with `raw_outputs/`, `models/`, and `tables/`.
5. Run `python3 scripts/validate_qpadm_fst.py` from this directory for structural cross-checks.

The underlying Human Origins genotype panel is not redistributed in this repository.
